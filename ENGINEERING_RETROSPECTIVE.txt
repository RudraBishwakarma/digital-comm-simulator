PURPOSE OF THIS DOCUMENT
================================================================================

The objective of this project was not simply to simulate digital
communication techniques.

Instead, the goal was to design a modular communication framework capable
of modeling the complete digital transmission chain used in modern wireless
communication systems.

Unlike textbook examples that isolate modulation or channel models, this
project was designed as an engineering platform where each subsystem
operates independently and communicates through well-defined interfaces.

Throughout development, emphasis was placed on

    • Modularity
    • Scalability
    • Engineering correctness
    • Mathematical validation
    • Software maintainability

rather than producing isolated simulation scripts.

Each engineering decision documented in this report follows the format

    CONTEXT
    DESIGN DECISION
    ALTERNATIVES CONSIDERED
    WHY THIS DESIGN WAS CHOSEN
    TRADE-OFFS
    FUTURE IMPROVEMENTS

1. WHY BUILD A COMMUNICATION SIMULATOR?
================================================================================

CONTEXT

Most undergraduate communication experiments focus on solving equations
or plotting modulation waveforms.

Although these exercises explain theoretical concepts, they rarely expose
how an actual communication system processes information from transmitter
to receiver.

The objective of this project was therefore to bridge the gap between
communication theory and practical implementation.

Instead of studying individual algorithms independently, the simulator
models the complete communication chain.

        Source

          ↓

      Modulator

          ↓

 Communication Channel

          ↓

     Demodulator

          ↓

    Performance Analysis

================================================================================
DESIGN DECISION
================================================================================

The simulator was designed to represent every stage of a practical
communication system.

Rather than writing isolated scripts for BPSK or QPSK, each communication
block was implemented as an independent subsystem.

================================================================================
ALTERNATIVES CONSIDERED
================================================================================

Alternative 1

Write one Python file for every modulation scheme.

Advantages

• Easy to understand

Disadvantages

• Difficult to extend

• Code duplication

• Poor maintainability

------------------------------------------------------------

Alternative 2

Develop one reusable communication framework.

Advantages

• Modular

• Easy to maintain

• Easily extensible

Chosen.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Real communication systems consist of independent processing blocks.

Separating these blocks allows new modulation techniques, channel models,
or receiver algorithms to be added without redesigning the entire project.

================================================================================
INTERVIEW DISCUSSION
================================================================================

The simulator demonstrates engineering thinking rather than simply
implementing mathematical equations.

3. DESIGNING THE COMMUNICATION PIPELINE
================================================================================

CONTEXT

Early prototypes directly connected modulation, channel, and receiver
logic inside one script.

As additional modulation schemes were planned, the code became difficult
to maintain.

================================================================================
DESIGN DECISION
================================================================================

The simulator follows a fixed communication pipeline.

Random Bits

↓

Modulator

↓

Communication Channel

↓

Receiver

↓

BER Calculator

↓

Visualization

Each subsystem communicates only through signal data.

================================================================================
WHY THIS DESIGN WORKS
================================================================================

Each module performs a single engineering task.

The channel module has no knowledge of modulation.

The receiver has no knowledge of bit generation.

The BER calculator operates independently of both.

This separation reduces coupling between subsystems.

================================================================================
TRADE-OFFS
================================================================================

Advantages

• Modular

• Easy debugging

• Easy testing

• Easy expansion

Disadvantages

• Slightly larger codebase

• More project structure

The increased maintainability outweighed the additional complexity.

5. SIGNAL REPRESENTATION
================================================================================

CONTEXT

Communication signals can be represented using

• Voltage

• Real numbers

• Complex IQ symbols

Modern wireless communication systems almost exclusively operate in the
complex baseband domain.

================================================================================
DESIGN DECISION
================================================================================

The simulator internally represents signals as complex-valued samples.

Example

s = I + jQ

instead of physical RF waveforms.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Complex baseband representation significantly simplifies simulation.

Carrier generation becomes unnecessary.

Phase information remains preserved.

Most DSP algorithms operate directly on IQ samples.

This representation is also used by

• LTE

• 5G NR

• Wi-Fi

• SDR

================================================================================
6. PROJECT DESIGN PHILOSOPHY
================================================================================

Several engineering principles guided development.

1. Every module should solve one problem.

2. Communication theory should remain visible inside the code.

3. Algorithms should be implemented manually whenever practical.

4. Simulation results must be validated using engineering metrics rather
than visual inspection.

5. The project should remain extensible toward LTE, OFDM, and Software
Defined Radio.

These principles influenced every design decision documented throughout
this retrospective.

7. RANDOM BIT GENERATION
================================================================================

CONTEXT

Every digital communication system begins with information generation.

Although manually entering binary sequences is sufficient for demonstrating
modulation, it does not accurately represent real communication systems
where millions of bits are continuously transmitted.

The simulator therefore required an automated source capable of generating
large statistically independent bit streams.

================================================================================
DESIGN DECISION
================================================================================

Implemented a pseudo-random binary source using

    numpy.random.randint()

instead of manually defining bit sequences.

The generator allows configurable transmission lengths ranging from a few
hundred bits to several million bits.

================================================================================
ALTERNATIVES CONSIDERED
================================================================================

Alternative 1

Static Bit Sequence

101010101...

Advantages

• Easy debugging

Disadvantages

• Poor statistical representation

------------------------------------------------------------

Alternative 2

Pseudo Random Binary Sequence (PRBS)

Advantages

• Realistic traffic

• Accurate BER measurement

• Unlimited packet size

Chosen.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Communication systems are evaluated statistically.

Performance metrics such as BER become meaningful only when calculated over
large random datasets.

Pseudo-random generation therefore better approximates practical
communication traffic.

================================================================================
ENGINEERING INSIGHT
================================================================================

Using random data also prevents algorithms from accidentally appearing
correct because they were tested only on favourable input patterns.

8. BPSK MODULATION DESIGN
================================================================================

CONTEXT

BPSK was selected as the first modulation technique because it represents
the simplest coherent digital modulation scheme.

It also serves as the mathematical foundation for more advanced modulation
methods.

================================================================================
DESIGN DECISION
================================================================================

Bits were mapped using bipolar symbols

        0 → -1

        1 → +1

instead of

        0 → 0

        1 → 1

================================================================================
ALTERNATIVES CONSIDERED
================================================================================

Binary Mapping

Advantages

• Simple implementation

Disadvantages

• DC Offset

• Incorrect symbol energy

• Incorrect BER characteristics

------------------------------------------------------------

Bipolar Mapping

Advantages

• Zero mean

• Unit energy

• Matches communication theory

Chosen.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Theoretical BER equations assume antipodal signaling.

Using bipolar symbols ensures

• Equal energy

• Symmetric constellation

• Zero average voltage

which produces communication behaviour consistent with analytical models.

================================================================================
TRADE-OFFS
================================================================================

No meaningful disadvantages.

Bipolar signaling is universally used in practical BPSK systems.

================================================================================
INTERVIEW DISCUSSION
================================================================================

A common interview question is

"Why is BPSK represented using ±1 instead of 0 and 1?"

Correct answer

The receiver performs maximum-likelihood detection around zero.

A zero-centered constellation minimizes decision errors.

================================================================================
9. IMPLEMENTING GRAY CODED QPSK
================================================================================

CONTEXT

After validating BPSK, the simulator was extended to support QPSK.

QPSK doubles spectral efficiency by transmitting two bits per symbol.

================================================================================
DESIGN DECISION
================================================================================

Implemented Gray Coding instead of natural binary ordering.

Constellation

             01        11

             ●────────●

             │        │

             │        │

             ●────────●

             00        10

================================================================================
ALTERNATIVES CONSIDERED
================================================================================

Natural Binary

Advantages

• Simple lookup table

Disadvantages

• Adjacent symbols differ by multiple bits

------------------------------------------------------------

Gray Coding

Advantages

• Adjacent symbols differ by one bit

• Lower BER

Chosen.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Noise typically moves received symbols to neighbouring constellation
points.

Gray Coding ensures neighbouring symbols differ by only one bit.

Consequently

Symbol Error Rate remains unchanged

while

Bit Error Rate decreases.

================================================================================
ENGINEERING INSIGHT
================================================================================

Gray Coding demonstrates an important communication engineering principle.

Reducing the number of incorrect bits is often more important than reducing
the number of incorrect symbols.

10. SYMBOL ENERGY NORMALIZATION
================================================================================

CONTEXT

Different modulation techniques naturally produce different average symbol
energies.

Without normalization, communication performance cannot be fairly compared.

================================================================================
DESIGN DECISION
================================================================================

Every modulation scheme is normalized to

Average Symbol Energy

Es = 1

before transmission.

Examples

BPSK

Already normalized.

QPSK

Divided by √2.

Future

16-QAM

64-QAM

will also be normalized.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Signal-to-Noise Ratio is defined relative to signal power.

If different modulation schemes transmit different energies, BER
comparisons become mathematically invalid.

Normalization guarantees every modulation operates under identical power
conditions.

================================================================================
ENGINEERING TRADE-OFF
================================================================================

Additional floating-point operations are introduced.

However, computational overhead is negligible compared to the benefit of
physically meaningful simulation results.

13. MODELING THE COMMUNICATION CHANNEL
================================================================================

CONTEXT

A communication system cannot be evaluated by connecting the transmitter
directly to the receiver.

In real-world wireless communication, transmitted signals experience
multiple impairments before reaching the receiver.

These include

    • Thermal Noise
    • Multipath Propagation
    • Attenuation
    • Interference
    • Phase Distortion

Ignoring these effects would make the simulator unrealistic.

================================================================================
DESIGN DECISION
================================================================================

The first implementation models the communication channel using the
Additive White Gaussian Noise (AWGN) channel.

Communication Pipeline

    Transmitter

          ↓

     AWGN Channel

          ↓

      Receiver

The channel modifies the transmitted signal by introducing controlled
Gaussian noise whose variance depends on the selected Signal-to-Noise
Ratio (SNR).

================================================================================
ALTERNATIVES CONSIDERED
================================================================================

Ideal Channel

Advantages

• Simplest implementation

Disadvantages

• Unrealistic

• BER always equals zero

------------------------------------------------------------

AWGN Channel

Advantages

• Industry standard benchmark

• Simple mathematical model

• Matches communication theory

Chosen.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Nearly every digital communication algorithm is first validated using an
AWGN channel before evaluating more complex environments such as Rayleigh
or Rician fading.

This provides a common baseline for comparing modulation schemes.

================================================================================
ENGINEERING INSIGHT
================================================================================

The AWGN channel represents the "hello world" of communication systems.

Every practical communication algorithm is expected to perform correctly
under AWGN before being tested under realistic propagation conditions.

================================================================================
14. SIGNAL-TO-NOISE RATIO IMPLEMENTATION
================================================================================

CONTEXT

Communication engineers specify channel quality using Signal-to-Noise
Ratio (SNR), typically expressed in decibels (dB).

However, Gaussian noise generators require linear power values rather than
logarithmic units.

================================================================================
DESIGN DECISION
================================================================================

Implemented standard logarithmic conversion

                SNR(dB)
SNRlinear = 10
                --------
                  10

The simulator automatically converts user-selected SNR values before
computing noise variance.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Maintaining SNR in decibel form provides an intuitive interface for users,
while internal linear conversion ensures mathematically correct channel
behavior.

This mirrors the methodology used in MATLAB Communication Toolbox and
professional DSP software.

================================================================================
TRADE-OFFS
================================================================================

Additional mathematical operations are introduced.

However, conversion is performed only once per simulation and therefore
adds negligible computational overhead.

15. GAUSSIAN NOISE GENERATION
================================================================================

CONTEXT

The quality of an AWGN simulator depends entirely on how accurately noise
samples follow the Gaussian probability distribution.

Random numbers alone are insufficient.

================================================================================
DESIGN DECISION
================================================================================

Noise samples are generated using

    numpy.random.normal()

Each sample has

Mean

μ = 0

Variance

σ² determined by SNR

The generated noise is then added directly to the transmitted signal.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Gaussian-distributed noise accurately models thermal noise generated by
electronic circuits and receiver front-end components.

It is the most widely accepted communication channel model.

================================================================================
ENGINEERING INSIGHT
================================================================================

Although communication channels appear random, they are governed by
well-defined probability distributions.

The simulator therefore focuses on statistical correctness rather than
arbitrary randomness.

================================================================================
16. RECEIVER DECISION ALGORITHM
================================================================================

CONTEXT

Recovering transmitted bits requires converting noisy received samples
back into discrete binary values.

This decision process determines overall communication performance.

================================================================================
DESIGN DECISION
================================================================================

Implemented Maximum Likelihood Detection.

For BPSK

Received Sample

> 0

→ Bit = 1

Received Sample

< 0

→ Bit = 0

For QPSK

Independent decisions are performed on

• In-phase Component

• Quadrature Component

================================================================================
ALTERNATIVES CONSIDERED
================================================================================

Fixed Arbitrary Threshold

Advantages

• Simple

Disadvantages

• High BER

------------------------------------------------------------

Maximum Likelihood Detection

Advantages

• Minimum probability of error

Chosen.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

Maximum Likelihood Detection is mathematically optimal under AWGN.

It minimizes decision errors while requiring minimal computational
complexity.

================================================================================
INTERVIEW DISCUSSION
================================================================================

A common interview question is

"Why is the BPSK threshold located at zero?"

Because the constellation is symmetric around zero and Gaussian noise is
equally likely to affect both symbols.

17. BIT ERROR RATE MEASUREMENT
================================================================================

CONTEXT

Visual inspection alone cannot determine communication quality.

A receiver may appear correct while silently introducing bit errors.

================================================================================
DESIGN DECISION
================================================================================

Implemented automatic BER calculation.

Original Bit Stream

↓

Receiver Output

↓

Bit-by-Bit Comparison

↓

Error Counter

↓

BER

The simulator reports BER after every simulation.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

BER is the primary metric used to evaluate digital communication systems.

Unlike waveform inspection, BER provides an objective quantitative measure
of receiver performance.

================================================================================
ENGINEERING INSIGHT
================================================================================

Modern wireless standards are designed around target BER values rather than
perfect transmission.

Engineering therefore focuses on acceptable error probabilities rather
than eliminating errors completely.

21. PERFORMANCE OPTIMIZATION USING NUMPY
================================================================================

CONTEXT

Communication simulations often process millions of transmitted symbols.

A straightforward implementation using Python loops quickly became the
largest computational bottleneck.

================================================================================
DESIGN DECISION
================================================================================

Vectorized numerical operations replaced explicit Python loops.

Instead of

for every symbol

↓

modulate

↓

transmit

↓

demodulate

The simulator processes complete arrays simultaneously.

================================================================================
WHY THIS DESIGN WAS CHOSEN
================================================================================

NumPy executes optimized C routines internally.

This significantly reduces interpreter overhead while improving execution
speed without changing mathematical behavior.

================================================================================
TRADE-OFFS
================================================================================

Advantages

• Faster execution

• Cleaner implementation

• Better scalability

Disadvantages

25. ENGINEERING LESSONS LEARNED
================================================================================

This project reinforced several important engineering principles.

1. Communication systems should be implemented as independent processing
blocks rather than monolithic scripts.

2. Correct signal normalization is essential for meaningful performance
evaluation.

3. Visualization is one of the most effective debugging techniques in Digital
Signal Processing.

4. Statistical validation is more reliable than validating individual
transmissions.

5. Software architecture becomes increasingly important as communication
systems grow in complexity.

6. Communication engineering combines mathematics, probability, signal
processing, and software engineering into a unified discipline.
